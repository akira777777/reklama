# reklama — Telegram-юзербот для рассылки рекламы

Одноразовый CLI-скрипт на Python + [Telethon](https://github.com/LonamiWebs/Telethon):
рассылает сообщение из `message.txt` (с медиафайлом из `media/`) по всем групповым
чатам аккаунта, где отправка возможна.

> ⚠️ **На свой риск.** Массовая рассылка от имени личного аккаунта нарушает правила
> Telegram и может привести к **бану аккаунта**. Скрипт применяет смягчения (только
> группы, большие случайные паузы, перерывы пакетами, окно активности, обработка
> FloodWait), но полной защиты не существует.

## Подготовка к работе

1. Получите `api_id` и `api_hash` на [my.telegram.org](https://my.telegram.org) в разделе *API development tools*.
2. Скопируйте шаблон переменных окружения:
   ```bash
   cp .env.example .env
   ```
   Откройте созданный файл `.env` и впишите ваши `TELEGRAM_API_ID` и `TELEGRAM_API_HASH`.
3. Положите медиафайл в `media/` (скрипт автоматически выберет первый файл в папке) и настройте текст рассылки в `message.txt`.

## Вариант 1. Запуск через Docker (Рекомендуемый)

Использование Docker позволяет запускать юзербот в изолированном окружении, избавляя вас от необходимости настраивать Python и устанавливать зависимости локально.

### 1. Сборка Docker-образа
```bash
docker compose build
```

### 2. Запуск скриптов
Поскольку Telethon требует интерактивного ввода при первом запуске (телефон, код из SMS, пароль 2FA), первый запуск **обязательно** нужно делать в интерактивном режиме с выделением терминала (это настроено по умолчанию для сервиса `app`):

```bash
# 1. Посмотреть, куда пойдёт рассылка (без отправки):
docker compose run --rm app python run.py --dry-run

# 2. Запустить полную рассылку (интерактивный вход при первом запуске):
docker compose run --rm app

# 3. Поиск групп с авто-вступлением:
docker compose run --rm app python search.py --query "маркетинг" --join --delay-min 15 --delay-max 30

# 4. Импорт сообщений из Saved Messages:
docker compose run --rm app python import_message.py
```

> [!NOTE]
> Файл сессии `reklama.session`, лог-файлы и файл прогресса рассылки `progress.json` будут автоматически сохраняться в вашей локальной папке на хосте благодаря настроенным Docker Volumes.

### 3. Проверки качества кода и тесты в Docker
Вы можете запускать все инструменты тестирования и статического анализа прямо в Docker-контейнере:

```bash
# Запуск тестов (PyTest):
docker compose run --rm test

# Проверка стиля и ошибок (Ruff):
docker compose run --rm lint

# Проверка типов (MyPy):
docker compose run --rm mypy
```

## Вариант 2. Локальный запуск (Без Docker)

Для локального запуска вам потребуется установленный **Python 3.11+**.

### 1. Установка окружения и зависимостей
```bash
python -m venv .venv
.venv\Scripts\activate          # Windows (PowerShell/CMD)
source .venv/bin/activate       # macOS / Linux

pip install -r requirements.txt # или requirements-dev.txt для разработки
```

### 2. Использование скриптов
```bash
# 1. Посмотреть, куда пойдёт рассылка (без отправки):
python run.py --dry-run

# 2. Дымовой тест на одной группе:
python run.py --limit 1

# 3. Полная рассылка (с resume по progress.json):
python run.py

# Сбросить прогресс и начать заново:
python run.py --reset-progress

# 4. Поиск публичных групп по ключевому слову (без вступления):
python search.py --query "маркетинг" --limit 10

# Поиск и автоматическое вступление в группы с задержками:
python search.py --query "маркетинг" --join --delay-min 15 --delay-max 30

# 5. Импорт сообщения (текст, медиа, разметка) из Saved Messages:
python import_message.py
```

### 3. Проверка качества локально (dev)
```bash
ruff check .
mypy .
pytest
```

## Умная рассылка и защита от банов

- **Spintax (рандомизация)**: В `message.txt` можно использовать шаблоны вида `{вариант1|вариант2|вариант3}`. Скрипт автоматически и случайно выберет один из вариантов для каждой отправки. Поддерживается вложенность, например `{Привет|Здравствуйте {друг|коллега}}`.
- **Резервная отправка текста**: Если отправка медиафайла завершается с ошибкой (файл не найден, сбой сети при выгрузке, неподдерживаемый формат), скрипт отправляет только текст, гарантируя доставку сообщения.
- **Адаптивные задержки**: При возникновении `FloodWaitError` скрипт переходит на динамическое увеличение пауз (коэффициент `delay_multiplier` возрастает до `1.5x`-`4.0x`), а после серий успешных доставок плавно снижает паузы до нормы.
- **Бесконечный FloodWait-повторитель**: Вместо пропуска чата при повторном лимите, скрипт ждет указанное Telegram время и пробует повторить отправку (до 5 раз), завершая план доставки.

## Настройка (`.env`, необязательно)

| Переменная | По умолчанию | Описание |
|---|---|---|
| `FORCE_DOCUMENT` | `false` | Всегда отправлять медиа как документ |
| `DELAY_MIN_SEC` / `DELAY_MAX_SEC` | `30` / `90` | Диапазон паузы между сообщениями |
| `BATCH_SIZE` | `50` | Размер пакета перед длинным перерывом |
| `BATCH_PAUSE_MIN_SEC` / `BATCH_PAUSE_MAX_SEC` | `300` / `900` | Перерыв между пакетами |
| `ACTIVE_HOURS` | (выкл.) | Окно активности, напр. `09:00-21:00` |
| `MEDIA_PATH` | первый файл из `media/` | Явный путь к медиа |

## Структура проекта

```
reklama/
  ├── auth.py             сессия Telethon, интерактивный логин
  ├── config.py           настройки + дефолты + переопределение из .env
  ├── dialogs.py          перебор диалогов, фильтрация (чистые функции)
  ├── emoji.py            парсинг и маппинг кастомных эмодзи в сообщениях
  ├── progress.py         чтение/запись progress.json, логика resume
  ├── sender.py           отправка сообщения/медиа, классификация ошибок
  └── utils.py            общие утилиты (логирование, очистка строк, менеджер сессий)
run.py                    CLI (argparse), оркестрация запуска рассылки
search.py                 поиск публичных групп и каналов, авто-вступление
import_message.py         импорт готовых сообщений с разметкой/медиа из чатов
Dockerfile                конфигурация Docker-образа
docker-compose.yml        описание Docker-сервисов (app, test, lint, mypy)
.dockerignore             файлы, исключаемые из Docker-контекста сборки
```

## Безопасность

- `.env` и `*.session` — это **секреты** (доступ к аккаунту). Они добавлены в `.gitignore` и полностью исключены из сборки Docker-образа.
- Не коммитьте и не передавайте их третьим лицам.
